Celestial Body – Cryptography Challenge

Category: Crypto

Objective: Decrypt a classified deep-space transmission encrypted with an AES key derived from a Truncated Linear Congruential Generator (LCG).

1. Initial Reconnaissance & The Red Herring

The provided transmission log includes multiple DSA signatures under the [TAP Authentication Signatures] section. Reviewing the encryption script reveals a glaring vulnerability in the TAP signing protocol: the DSA nonce (k_tap) is generated outside of the signing loop.

Because the exact same nonce is used for all five signatures, an attacker could easily recover the private signing key (x_tap) using standard DSA nonce reuse equations. However, this is a cleverly disguised red herring. The AES key used to encrypt the flag is completely independent of the TAP signing key.

2. Cracking the Epoch Time

The LCG parameters multiplier (a) and increment (b) are deterministically generated based on the exact position of Halley's Comet at the time of transmission. We are given the date (2026-01-26) but not the exact time.

However, the transmission log includes epoch_hash = 8b156702c993b9b5, which is the first 16 characters of the SHA-256 hash of the "HH:MM:SS" string. Since there are only 86,400 seconds in a 24-hour period, we can bruteforce the exact time: 

import hashlib

target_hash = "8b156702c993b9b5"
for h in range(24):
    for m in range(60):
        for s in range(60):
            t_str = f"{h:02d}:{m:02d}:{s:02d}"
            if hashlib.sha256(t_str.encode()).hexdigest()[:16] == target_hash:
                print(f"[*] Found Transmission Time: {t_str}")
                exit()

Result: The transmission occurred at exactly 04:12:55.

3. The Truncated LCG Exploit

With the exact time, we can run the challenge's load_comet_and_derive function to obtain LCG parameters a and b. The LCG has a 512-bit state, but the spacecraft only broadcasts the top 192 bits at specific step intervals (steps 0, 4, 10, 18, and 28). This is a classic Hidden Number Problem solvable via LLL lattice basis reduction.

Modeling the Mathematics

We know the mathematical relationship between the initial state (S0) and any future state (Sk): 

S_k ≡ A_k · S_0 + B_k  (mod p)

Where:
  A_k = a^k mod p
  B_k = b · (a^k − 1) · (a − 1)^{−1} mod p

Each state S is composed of our known telemetry (t) shifted by 320 bits, plus the unknown lower bits (x): 

S_k = t_k · 2^320 + x_k

Substituting and rearranging to isolate the unknowns on one side gives: 

x_k − A_k · x_0 ≡ K_k  (mod p)

Where K_k = A_k · t_0 · 2^320 + B_k − t_k · 2^320  (all known values)

Lattice Construction

We construct a 6×6 matrix in SageMath to model these relations. The matrix enforces the modulo p arithmetic and applies the A_k multipliers. Running LLL on this lattice finds the shortest basis vectors, and because the unknown lower bits are strictly bounded by 2320, LLL successfully isolates them — recovering the exact x0.

4. Final State Recovery & Decryption

By adding the recovered x0 to the shifted telemetry t0, we perfectly reconstruct the 512-bit initial state S0. The challenge generates the AES key by advancing the LCG exactly one step past the final index 28 (to step 29). We fast-forward to step 29, hash the result with SHA-256, and decrypt the ciphertext using AES-CBC.

def load_comet_and_derive(year, month, day, hour, minute, second):

    from skyfield.constants import GM_SUN_Pitjeva_2005_km3_s2 as GM_SUN

    with load.open('CometEls.txt') as f:

    row = comets.loc['1P/Halley']

    ts = load.timescale()
    t = ts.utc(year, month, day, hour, minute, second)

    eph = load('de421.bsp')
    sun = eph['sun']
    halley = sun + mpc.comet_orbit(row, ts, GM_SUN)

    astrometric = sun.at(t).observe(halley)
    x, y, z = astrometric.position.au

    coord_string = f"{x:.10f}_{y:.10f}_{z:.10f}"
    h_a = hashlib.sha512((coord_string + "_A").encode()).digest()
    h_b = hashlib.sha512((coord_string + "_B").encode()).digest()
    return bytes_to_long(h_a), bytes_to_long(h_b)
