1. ECDSA & LCG Background

In ECDSA, signing a message with hash e using private key d requires a random nonce k. The signature pair (r, s) is computed as: 

R = k·G  →  r = Rx mod n
s = k⁻¹ · (e + d·r) mod n

Security depends entirely on k being unpredictable. Here, nonces are produced by an LCG: 

k_{i+1} = a · k_i + b  (mod n)

Because a is known, three consecutive nonces satisfy a relationship that allows b to be eliminated entirely: 

k₂ = a·k₁ + b  (mod n)
k₃ = a·k₂ + b  (mod n)

Subtracting:  k₃ − k₂ = a·(k₂ − k₁)  (mod n)
Therefore:    k₃ = (a+1)·k₂ − a·k₁   (mod n)

3. Deriving a Linear Equation in d

Each nonce can be expressed in terms of the private key using the ECDSA equation: 

kᵢ = sᵢ⁻¹ · (eᵢ + d·rᵢ)  (mod n)

Let xᵢ = sᵢ⁻¹ mod n
Then: kᵢ = xᵢ·eᵢ + xᵢ·rᵢ·d  (mod n)

Substituting into the LCG relation k₃ = (a+1)·k₂ − a·k₁ and expanding: 

x₃(e₃ + d·r₃) = (a+1)·x₂(e₂ + d·r₂) − a·x₁(e₁ + d·r₁)

Collecting d terms:

d · [x₃·r₃ − (a+1)·x₂·r₂ + a·x₁·r₁] = (a+1)·x₂·e₂ − a·x₁·e₁ − x₃·e₃

Let:  A = x₃·r₃ − (a+1)·x₂·r₂ + a·x₁·r₁  (mod n)
      B = (a+1)·x₂·e₂ − a·x₁·e₁ − x₃·e₃  (mod n)

Then: d = B · A⁻¹  (mod n)

This is a single modular linear equation with one unknown — no lattice reduction, no brute force.

    "6464c3b1798b92c6831384a96e8e6f6b...3ef516d1fc848cf1"

modinv = lambda a, m: pow(a, m - 2, m)

x1, x2, x3 = modinv(s1, n), modinv(s2, n), modinv(s3, n)

cipher = Cipher(algorithms.AES(key), modes.ECB(), backend=default_backend())
pt = cipher.decryptor().update(ciphertext)
print(pt.decode())
